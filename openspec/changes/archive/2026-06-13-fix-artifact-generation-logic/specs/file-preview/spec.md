# Delta: file-preview

## MODIFIED Requirements

### Requirement: 生成ファイルを簡易編集できる

ユーザーは生成されたファイルの内容をブラウザ上のテキストエディタで編集・保存できる。編集を保存したファイルは編集済みとして記録され（`is_edited`）、一覧およびプレビュー上で編集済みであることが識別できる。

#### Scenario: ファイルの編集
- **WHEN** ユーザーがプレビュー画面で「編集」ボタンを押す
- **THEN** テキストエディタが表示され、ファイル内容を編集できる

#### Scenario: 編集の保存
- **WHEN** ユーザーが編集内容を保存する
- **THEN** `PUT /api/v1/projects/{id}/files/{file_id}` が呼ばれ、更新された内容と編集済みフラグがgenerated_filesテーブルに保存される

#### Scenario: 編集のキャンセル
- **WHEN** ユーザーが編集をキャンセルする
- **THEN** 変更は破棄され、元の内容が表示される

#### Scenario: 編集済みファイルの識別表示
- **WHEN** 編集済みのファイルが存在するプロジェクトの詳細ページを表示する
- **THEN** 該当ファイルに編集済みであることを示す表示（バッジ等）が付く

## ADDED Requirements

### Requirement: 編集済みファイルが存在する状態での再生成は確認を求める

フロントエンドは、編集済みファイルが存在するプロジェクトで再生成を実行する際、編集内容を保護したまま再生成するか、テンプレート出力で上書き（`force`）するかをユーザーに確認しなければならない（MUST）。

#### Scenario: 再生成時の確認ダイアログ
- **WHEN** 編集済みファイルが存在する状態で再生成ボタンを押す
- **THEN** 編集済みファイルの扱い（保護 / 上書き）を選択する確認が表示され、選択に応じた再生成が実行される

#### Scenario: 編集済みファイルがない場合
- **WHEN** 編集済みファイルが存在しない状態で再生成ボタンを押す
- **THEN** 確認なしで通常の再生成が実行される
