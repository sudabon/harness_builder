## ADDED Requirements

### Requirement: ウィザード分割後も既存フローを維持する

システムは、プロジェクト作成ウィザード画面を複数コンポーネントへ分割しても、既存の4ステップ入力、プリセット反映、バリデーション、回答保存、ファイル生成、生成後遷移の挙動を維持しなければならない（MUST）。

#### Scenario: ステップ表示と入力保持の互換性
- **WHEN** ユーザーがウィザードで値を入力し、次のステップへ進んだ後に「戻る」ボタンで前のステップへ戻る
- **THEN** 分割前と同じステップ表示に戻り、入力済みの値が保持されている

#### Scenario: プリセット反映の互換性
- **WHEN** ユーザーがプリセットを選択して確認ステップまで進む
- **THEN** 確認画面にはプリセット由来の言語、フレームワーク、テスト手法、Lint / Format の値が表示される

#### Scenario: 必須項目バリデーションの互換性
- **WHEN** ユーザーが必須項目を未入力のまま必須項目ステップで「次へ」ボタンを押す
- **THEN** 分割前と同じくバリデーションエラーが表示され、任意項目ステップへ遷移しない

#### Scenario: 生成実行の互換性
- **WHEN** ユーザーが確認ステップで「ファイル生成を実行」ボタンを押す
- **THEN** 回答保存とファイル生成が実行され、生成されたプロジェクト詳細ページへ遷移する
