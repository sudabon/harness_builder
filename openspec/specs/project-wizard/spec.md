# Capability: project-wizard

## Purpose

Harness Builder のユーザーが、ステップ形式のウィザードを通じてプロジェクト要件を入力し、後続のファイル生成に必要な情報を保存できるようにする。

## Requirements

### Requirement: ウィザード形式でプロジェクト情報を入力できる

ユーザーはステップ形式のウィザードを通じてプロジェクト情報を入力する。ウィザードはステップごとに関連する質問をグループ化して表示し、前後のステップへ自由に移動できる。

#### Scenario: ウィザードの開始
- **WHEN** ユーザーがプロジェクト新規作成ページにアクセスする
- **THEN** ウィザードの最初のステップが表示される

#### Scenario: ステップ間の移動
- **WHEN** ユーザーが現在のステップの入力を完了し「次へ」ボタンを押す
- **THEN** 次のステップに遷移する

#### Scenario: 前のステップに戻る
- **WHEN** ユーザーが「戻る」ボタンを押す
- **THEN** 前のステップに戻り、入力済みの値が保持されている

### Requirement: 必須項目を入力できる

システムは以下の必須項目の入力フォームを提供する：
- プロジェクト種別（Web / API / OSS / SaaS）
- 使用言語（Python / TypeScript 等）
- フレームワーク（FastAPI / React / Next.js 等）
- 使用AIツール（Claude / Codex / Cursor）— 複数選択可
- テスト手法（pytest / jest 等）
- Lint / Format（ruff / eslint / prettier 等）
- 禁止事項（自由記述テキスト）
- レビュー方針（厳格 / 柔軟）

#### Scenario: 必須項目が未入力の場合
- **WHEN** ユーザーが必須項目を未入力のまま「次へ」を押す
- **THEN** バリデーションエラーが表示され、次のステップに遷移しない

#### Scenario: すべての必須項目を入力
- **WHEN** ユーザーがすべての必須項目を入力して「次へ」を押す
- **THEN** 次のステップまたは確認画面に遷移する

### Requirement: 任意項目を入力できる

システムは以下の任意項目の入力フォームを提供する：
- ブランチ戦略
- CIコマンド
- デプロイ制約
- セキュリティ要件
- 過去の失敗事例
- 命名規約

#### Scenario: 任意項目を空のまま進む
- **WHEN** ユーザーが任意項目を空のまま「次へ」を押す
- **THEN** バリデーションエラーなく次のステップに遷移する

#### Scenario: 任意項目を入力する
- **WHEN** ユーザーが任意項目に値を入力する
- **THEN** 入力値が保存され、ファイル生成時に反映される

### Requirement: 回答をAPIに保存する

ウィザードの回答はバックエンドAPIに送信・保存され、後続のファイル生成処理で利用される。

#### Scenario: 回答の保存
- **WHEN** ユーザーがウィザードを完了し「生成」ボタンを押す
- **THEN** 回答が `PUT /api/v1/projects/{id}/answers` に送信され、questionnaire_answersテーブルに保存される

#### Scenario: 回答の更新
- **WHEN** ユーザーがウィザードに戻って回答を変更し再度保存する
- **THEN** 既存の回答が更新される
